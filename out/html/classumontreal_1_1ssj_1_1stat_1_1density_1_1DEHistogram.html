Detailed Description

This class provides methods to construct, manipulate, and evaluate a histogram density estimator for univariate densities.

The construction of a histogram is based on a set of \(n\) individual observations \(X_0, …, X_{n-1}\) which, in turn, can be realizations of a umontreal.ssj.mcqmctools.MonteCarloModelDouble, for instance. In any case, this type of density estimator can by built from pure data stored in an array, form a umontreal.ssj.stat.TallyHistogram, or even from a umontreal.ssj.stat.ScaledHistogram.

It is essential to confine oneself to a finite interval \([a,b]\). For a fixed number of bins \(s>0\) we partition the interval \([a,b]\) into \(s\) subintervals of equal lengths \(h\). Observe that \(h=(b-a)/s\) and, hence, histogram estimators can also be parameterized by a bin width \(h>0\), as long as the resulting number of bins is an integer. The estimator itself is then defined by

\[ \hat{f}_{n}(x) = \hat{f}_{n,h}(x) = \frac{n_j}{nh},\quad\text{for } x\in[a+(j-1)h, a+jh), \quad j=1,\dots,s, \]

where \(n_j\) denotes the number of observations that fall in this interval. Observe that changing \([a,b]\), \(h\), or \(s\) changes the structure of the histogram completely. Thus, after any alteration of these parameters the histogram needs to be constructed afresh.

As histograms are constant within one bin certain quantities, such as the variance, are not affected by shifting an evaluation point within its bin. Hence, selecting one evaluation point per bin is sufficient for such methods. To this end, this class provides methods to evaluate the density estimator once in each bin, without having to explicitly specify an array of evaluation points.

It needs to be added that, due to the fact that the partition into \(s\) bins relies on half-open intervals, the boundary \(b\) is not included in any of these intervals. Since the probability of an observation being exactly equal to \(b\) is zero, we can effectively ignore this subtlety.
